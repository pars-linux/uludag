Translating Pardus

A language translation consists of translation of many modules which we call "po files". The po files can be translated by Kbabel (under Linux) or Poedit (under Windows). Kbabel can be installed from Pardus repository under the name "kdesdk".

First priority

The 'first priority' items below are considered as the major sources which should be translated at least 80% for a language to be available with Pardus.

  • Localization of installer (YALI) including the user interface and help files (language file)
  • Localization of help and user interface files of the following Pardus software:
    • Network manager (language file)
    • Package manager (language file)
    • Feedback (language file)
    • Firewall configurator (language file)
    • Tasma (language file)
    • Service manager (language file)
    • User manager (language file)
  • Localization of Mudur, the init system of Pardus (language file)
  • Localization of ÇOMAR, the configuration manager (language file)
  • Localization of sysinfo, system information center (language file)
  • Localization of Kaptan, first-run application (language file)

Pardus uses KDE for the primary desktop of own. For this reason, the translation percent of your language for KDE should be over 80% in order to be supported for Pardus.

Second priority

These are second priority items. Note that you don't have to translate them, however doing so is a big plus.

  • Release notes: Each Pardus release has its own release notes which is published on the internet and put on the CD.
  • Summary/description localization of specification files. You can see the status of this translation under the label "Package descriptions" in statistics page.

Applying for translation

  • Sign up for a developer account, as mentioned here. To commit a translation, you need to know how to use a Subversion system.
  • After you get a developer account from Pardus admins, request for a mailing list from admins@pardus.org.tr. Your language mailing list is in the form of pardus-translation-XX where XX is the ISO code of your language (i.e nl for Dutch, it for Italian etc).
  • Translate the language po files to your language and commit them in the respective repository. Currently, all of the po files mentioned above are in Uludag repository.
  • While committing, modify the name of the language file which resembles your language (i.e for German translation for the Feedback application, put it as feedback/po/de.po in Subversion).
  • Use the statistics to have a brief overview of the status of your language periodically.
