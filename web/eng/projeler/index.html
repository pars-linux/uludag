COMAR

COMAR (COnfiguration MAnageR), is configuration manager that makes softwares compatible with other softwares installed on system. It stores, which tasks are capable to be handled and which tasks and other relevant informations are required by each application. With the ÇOMAR assistance, applications are able to set up themselves according to existence and skills of the others.

To access this projects site, please click here.

PISI

PISI (Packages Installed Succesfully as Intented) is the package management system of Uludağ project. It provides installing/removing the packages to/from the system without any error. It stores package dependencies to other packages/libraries and COMAR tasks. It finds out package(s) to handle COMAR tasks, like opening a file type or serving web pages, and handles the job requestes of users, without having prior knowledge of the name or any other detailed information about his/her requirement.

YALI

YALI (Yet Another Linux Installer) is operating system installer system that installs packages selected by user from installation media (eg. CD, network, Internet) to target system, quickly and without any error.

Packages

"Package" is the name of a file that is specifically arranged for installing/removing applications to/from the system easily and independently. The Packages project is responsable for determining which packages take place in Uludag, and forming the source code of those packages with taking in to account Uludag objectives.

Application selection, error correction, resolving problems peculiar to Turkish language, gathering information needed by PISI and COMAR and packaging in those information, are all in scope of this project.
