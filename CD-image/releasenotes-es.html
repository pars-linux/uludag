Notas de release Pardus 2007

1. Bienvenido a Pardus 2007

Le damos la bienvenida a Pardus versión 2007, con el cual va a encontrar un sistema operativo fácil de instalar y fácil de usar, con una serie de componentes útiles, seguros y compatibles desarrollados para el escritorio KDE. Con Pardus puede conectarse a Internet, leer sus mensajes de email, comunicar de modo segura con sus amigos, aprovechar las facilidades multimedia de su PC escuchando musica y observando peliculas. Le invitamos a visitar visit www.pardus.org.tr/eng para descubrir como Pardus puede responder a sus necesidades en computación.

1.1 Feedback

Puede utilizar bugs.pardus.org.tr, alias "Pardus bug tracking system" para enviar reportes de fallas, correcciones de errores, o cualquier sugerencia relacionado a Pardus. Le agradecemos por su interés y amabilidad al apoyarnos con su aporte de reportes, comentarios y sugerencias para mejoras y paquetes a incluir para la proxima versión.

Tambien puede utilizar el aplicativo de feedback, que está incluido en Pardus 2007, para nviar sus ideas sobre Pardus por la internet.

2. Características principales

Esta sección describe las características de Pardus 2007.

2.1 Usabilidad de escritorio (KDE 3.5.5)

E escritorio KDE, versión 3.5.5, que está incluido en Pardus 2007, cumple los requisitos y expectativas delos usuarios de PC con esquemas de color, diseño de menú, soporte de localización para más de 65 idiomas, acceso rápido a aplicaciones multimedia, conexión fácil a internet y mucho más. Alrededor de 1500 mejoras mayores y menores se implementaron con esta versión de KDE. La particularidad más destacada es la reducción de tiempo de instalación por 20%. Además hay caracteristicas mejoradas en los compoonentes KMail, Akregator, Kopete y KPDF.

2.2 Comunicación y compatibilidad con Windows

Pardus 2007 es compatible para interactuar con otros sistemas operativos en la red. Puede aprovechar discos rígidos, impresoras y otros  periféricos compartidos en windows. De la misma manera Pardus 2007 puede compartir sus discos para windows.

2.3 Soporte Java

La plataforma de de desarrollo Sun Microsystems Java está incuido en Pardus 2007. Aplicaciones basados en Java, independientes del sistema operativo, pueden ser ejecutados. Tambien es posible acceder directamente a las amplias librerías de Java por todo el mundo.

2.4 Juegos y entretenimiento

Pardus 2007 provee una serie de juegos para que se puede divertir en su tiempo libre. Además puede instalar y jugar un montón más, descargandolos de los repositorios de Pardus con pocos clicks.

2.5 Servicios de Red

Pardus puede ser utilizado para proveer los siguientes servicios de red:
  • Servidor de archivos (Samba)
  • Servidor de impresión (Cups)
  • Servidor Web (Apache)
  • E-mail (Postfix)
  • Servidor FTP (Vsftpd)
  • Servidor OpenLDAP (Openldap software)

3. Requerimientos de Installación

Pardus puede ser ejecutado con un hardware promedio. Sugerimos un mínimo de 512 MB RAM (memoria) y procesador de 1200 MHz para un rendimiento efectivo. Los requerimientos mínimos para ejecutar Pardus se mencionan abajo:
  • 256 MB memoria (sugerimos 512 MB)
  • 800 MHz procesador Intel o AMD (sugerimos mínimo 1200 MHz)
  • 4 GB espacio en disco rígido

4. Mejoras en esta versión

Pardus 2007 fue lanzado luego de un peródo de desarrollo de más de 6 meses. Esta versión incluye mejoras a base de sugerencias de usuarios, como detallado abajo.

4.1 Cortafuegos Pardus

Cortafuegos Pardus fue diseñado como software de uso fácil, con la finalidad de proteger contra ataques desde las afueras del sistema. Corre continuamente en el sistema para proteger contra posibles ataques. Mediante el cortafuegos Pardus puede deshabilitar conexiones entrantes a la PC, bloquear un puerto específico, o monitorear las conexiones actuales.

4.2 Sistema de administración de paquetes mejorado

PISI ahora usa el algoritmo LZMA, resultando en una mejora de velocidad de 30%, y ahorrando espacio de las aplicaciones. Con esta mejora  los paquetes PISI y los software a descargar por el usuario disminuyen en tamaño, lo que posibilita distribuir más software con Pardus 2007.

4.3 Nueva interfaz de la administracón de software

interfaz del administrador de paquetes PISI sustituido por su sucesor de uso fácil (package manager). Este nuevo software permite de manera fácil instalar, desinstalar o actualizar programas.

4.4 Mejoras en sistema de instalación YALI

Algunas de las fallas fueron corregidos en YALI y está configurado para usarse en conjunto con la infraestructura de COMAR y PISI. Además, la interfaz de YALI fue ajustada y está con colores cambiados.

4.5 Nuevo y mejorado sistema de arranque de Pardus

Los procesos entre el arranque del kernel, hasta la pantalla de login son controlados por el sistema de arranque. Esto incluye el montaje del sistema de archivos, detección de la información del hardware e instalación de sus drivers, configuración de red básico, y arranque de servicios del sistema. Tambien es responsable de parar los servicios, desmontar el sistema de archivos y manteniendo su integridad, al apagar la computadora.

La fuente del sistema de arranque de Pardus fue re-escrito desde zero, porque el sistema de arranque de Pardus 1.0 era demasiado lento y su  capacidad de detección de hardware era insuficiente. El nuevo sistema viene con "mudur" ("Gerente") un script básico de arranque, hecho en Python, y "muavin" ("asistente"), tambien escrito en Python, responsable de buscar por los módulos de hardware e instalarlos. Los scripts de los servicios del sistema son controlados por Comar, con la ventaja de procesamiento en paralelo y de uso fácil. Mediante esta mejora se logra,
  • Reducción significante de tiempos de arranque y de apagado.
  • Pardus 2007 se volvió una distribución más flexible, y fácil de mantener con el poder de Python.
  • Servicios son diseñados y programados para poder controlarlos desde linea de comando, o con la interfaz de TASMA.

4.6 Más detección de hardware

El nuevo sistema de identificación de hardware, peculiar de Pardus, permite una mayor gama de hardware del mercado y funciona harmónicamente con ellos. Reduce la necesidad de identificación manual de hardware, por lo tanto el usuario final no necesita configurar los  drivers. Pardus puede identificar y configurar el hardware automaticamente durante el arranque del sistema.

El sistema de detección de hardware, con nombre Muavin, evita la carga de módulos innecesarios optimizando así el uso de la memoria. Además, en vez de usar la tecnología anterior del sistema hotplug, usa udev, que es una tecnología más nueva y más flexible para nombramiento dinámico de dispositivos.

4.7 Corregidos más de 1200 fallas

Más de 1200 reportes de fallas del sistema de seguimiento de fallas fueron examinados y cuidadosamente corregidos por los desarrolladores, desde release 1.0, así dejando lo más estable y adaptado a las necesidades de los usuarios finales.

4.8 Nuevo kernel 2.6.18

La mayoría de los problemas de hardware solucionado con el último kernel 2.6.18 proveído con Pardus. Soporte para modems USB ADSL permite identificar más USB ADSL modems que la versión anterior. Además soluciona un problema de arranque, causado por APIC/ACPI en maquinas antiguos. El tiempo de arranque del kernel fue reducido a 3 secundos con un parche de identificación de discos rígidos IDE.

Algunos de los nuevos hardware soportados:
  • Nuevos dispositivos SATA
  • Soporte USB ADSL modem 
  • Soporte mejorado para redes inalambricas
  • Soporte para más camaras, escáners e impresoras

4.9 Nuevos íconos

Una de las características de mejor aceptación con Pardus 1.0 fueron los juegos de íconos "Tulliana". Estos juegos fueron mejorados con Pardus 2007 y se incluyeron íconos adicionales. Nombrado Tulliana 2.0 icon set, está ahora más conforme con el escritorio de Pardus y las demás aplicaciones, y así aporta para que puede disfrutar su tiempo usando el escritorio.

4.10 Programa de administración de usuarios renovado

En versión 2007, "user-kga" cambió a "user manager", que puede ser usado para agregar un nuevo usuario a Pardus. Esta nueva herramienta permite una administración fácil de derechos de usuarios, como uso de CDs, escuchar musica y jugar juegos. De manera fácil uno puede asignar funciones de usuarios. Un usuario con derechos de administrador se convierte en administrador y puede realizar areas de administración, sin necesidad de saber la contraseña de root.

4.11 Nueva aplicación: Service manager

Service manger viene con Pardus 2007 para administrar fácilmente los servicios y los servicios que se inician durante el booteo. Con este administrador de servicios se configura los servicios a iniciar durante el arranque, además se puede arrancar o pararlos cuando necesario.

4.12 Actualizaciones de software (xorg, gcc, ..)

Con Pardus 2007, muchas fallas relacionados con Xorg fueron corregidos e incluye actualizaciones importantes para tarjetas gráficas. Incluido en Pardus 2007 está también la ultima release estable 3.4 del compilador GNU C (gcc) . Esta versión incluye bastante buenas caracteristicas para programadores de software en C++. Glibc y paquetes binutils se compilan con los parches nuevos (-Bdirect, -hashvals, -dynsort), por lo tanto se espera una aceleración del arranque de los programas.

4.13 Mejoras en Skype, Kopete y otras herramientas de comunicación

Skype es una solución completa de telefonía IP usando el protocolo H.323. uede hacer llamadas de teléfono con Pardus usando Skype. Además puede anexar una webcam y realizar videoconferencias con usuarios usando Kopete.

4.14 Soporte de localizaci+on

Pardus 2007 ahora incluye soporte para  inglés, turco, alemán y holandés desde la instalación hasta el escritorio. Soporte para idiomas adicionales será disponible más adelante.

4.15 Nueva release de OpenOffice.org 2.0.4

Esta nueva release de OpenOffice.org incluye muchas correciones de código y tambien provee una mejora de 23% de performance en la aplicación Calc. Mejora la compatibilidad con tipos de archivos de MS Office y tiene falilidades adicionales en la generación de documentos pdf. Otra facilidad de este release es la compatibilidad mejorada MS Basic en hojas de cálculo y acceso de lectura para bases de datos MS Access.

4.16 Nueva granja de servidores

Con Pardus 2007 todos los paquetes PISI se preparen en una granja de servidores, de los cuales se los puede descargar más velozmente. Build farm consiste de 2 maquinas potentes y su capacidad de procesamiento está a disposición de todos los desarrolladores de Pardus. Con cada release de un paquete, inicia una fase de testeo por el equipo de testeo de Pardus, corrigiendo posibles errores, para luego agregarlos al ciclo de actualizaciones.
