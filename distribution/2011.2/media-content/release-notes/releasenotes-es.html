Notas de Lanzamiento

1. Welcome to Pardus 2011.2 Cervus elaphus

Welcome to Pardus 2011.2 Cervus elaphus, the latest maintenance release of Pardus, a powerful, KDE-focused Linux distribution.

1.1. Qué es Pardus ?

Pardus es un proyecto de una distribución de GNU/Linux amigable al usuario, creado con código abierto, a libre distribución, y liecnciado pajo la licencia GPL (General Public License) , con varios mejoras en usabilidad, compatibilidad de hardware, estabilidad y seguridad.
El auspiciante (sponsor) del proyecto Pardus es el Centro de Investigación de Tecnologías Avanzadas de Informática y Seguridad Informática (BILGEM), un asociado del Concilio de Investigación Científica y Tecnológica de Turquía (TUBITAK).
La distribución Pardus se esfuerza para brindar a la gente una experiencia de computación estable, confiable y fácil de usar mediante la provisión de software de código abierto y popular. Además los desarrolladores trabajan arduamente para lograr una compatibilidad con una amplia gama de hardware.
El nombre del proyecto viene del nombre en latín (cientifico) del leopardo de Anatolia, Panthera pardus tulliana.

1.2. Como puedo probar Pardus ?

Para usar Pardus puede instalarlo en su computadora o ejecutando el sistema en la versión live (en vivo) sin modificar su disco duro. Puede bootear Pardus con un lector óptico o de un dispositivo de almacenamiento USB como discos externos o Pendrive.
Tanto la versión liva como la versión de instalación superan el tamaño de una CD común, por lo tanto si prefiere utilizar un medio óptico para bootear Pardus, debe grabar las imagenes en DVD.
Tenga en cuenta que debe asegurarse que en el orden de arranque en el BIOS el lector óptico o el dispositivo USB deben estar en primer posición para arrancar de ello tanto para la instalación como en modo vivo. En algunos sistemas se puede seleccionar el orden de dispositivos de arranque apretando una tecla (generalmente F8 para equipos de escritorio) durante el arranque.

4. Requirimientos al Sistema

4.1. Procesador (CPU)

Le convendría instalar la versión de 64-bit de Pardus 2011.2 si sus sistema está equipado con uno de los procesadores del listado abajo.

4.1.1. Procesadores INTEL con soporte a la arquitectura 64-bit

  • Versiones más reciente de Pentium 4, Celeron D, Xeon y Pentium Dual-Core
  • Atom D5xx, N4xx and N5xx
  • Todas las versiones de Pentium D, Pentium Extreme Edition, Core 2, Core i7, Core i5 y Core i3
  • Probablemente procesadores con fabricación más reciente que este informativo.

4.1.2. Procesadores AMD con soporte a la arquitectura 64-bit

  • AMD Athlon 64, 64 X2, 64 FX
  • AMD Athlon II
  • AMD Opteron
  • AMD Turion 64
  • AMD Turion 64 X2
  • AMD Sempron
  • AMD Phenom
  • AMD Phenom II
  • Probablemente procesadores con fabricación más reciente que este informativo.

4.2. Memoria del sistema (RAM)

Los requirimientos de memoria RAM de Pardus 2011.2 dependen del tipo de uso que le dan al sistema, y por lo tanto es dificil de determinar límites inferiores o superiores. Sin embargo recomendamos al mínimo 1GB de RAM para una experiencia agradable del escritorio.

4.3. DIsco rígido

4.3.1. Requisitos mínimos

Pardus 2011.2 requiere al menos 4GB de espacio en disco para uns instalación exitosa. Este valor de espacio está predeterminado por la cantidad de paquetes que se van a instalar, ya que durante de la instalación no se tiene la opción de elegir los paquetes a instalar.

4.3.2. Requirimientos recomendados

Para un uso eficiente de Pardus 2011.2 en su sistema sin llegar al límite de espacio en disco para su uso diario, recomendamos al menos 10GB de espacio disponible en disco.

4.3.3. Comentario sobre espacio Swap

El sistema usa el espacio de intercambio (Swap), cuando la memoria RAM del sistema ya no da abasto. Si cree que tiene suficiente memoria RAM para su uso diario, puede omitir la creación del espacio de intercambio (swap) durante la instalación.
Atención: Si desea utilizar la funcionalidad de hibernar (Suspend to disk) en Pardus se necesita la existencia del espacio de intercambio Pardus. Esta función queda deshabilitada si no existe el espacio de intercambio.
