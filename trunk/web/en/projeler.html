COMAR

COMAR (COnfiguration MAnageR), is a configuration framework which will function as a middle layer between package configurations. Packages will tell the jobs they can handle to the COMAR. They also ask the jobs or information they need from other packages from COMAR. These jobs would be specified within the COMAR's object model (OM) as a hierarchical class tree.

Nodes of the OM tree would be encapsulations of specific jobs which are required for integration between the packages. For example, Desktop.Menu class would be provided by the KDE panel application, and other packages would give their menu entries to this object. Another example would be a Network.Transmission class provided by local system's kernel iptables, and gateway's firewall application. When Apache installed on such a system, both local firewall and gateway will open the web connection (port 80) to the Apache.

Since modifying the applications themselves is hard and would tie them to COMAR too much, we are going to use scripts written in a custom language called COMAR Scripting Language (CSL). These scripts will come with the package and provide the COMAR objects. Upon a call from COMAR, they'll handle modifying package's configuration files according to the request, sending signals to the application or restarting it when necessary.

There would be a COMAR daemon for keeping track of the requests, handling atomicity of transactions, retrying them when they failed for temporary problems. It will also make it easy to implement remote objects, and Access Control Lists (ACLs) over local and remote objects.

PISI

PISI (Packages Installed Succesfully as Intented) is our package management system. It will handle installing and removing packages from network or installation media. PISI will be based on COMAR. Dependencies could be either packages/libraries or COMAR classes. It is also possible to request a package by classes it provides. Thus a user will be able to install a program for handling a MIME type or serving emails without knowing the names of required package names.

YALI

YALI (Yet Another Linux Installer) is our installer. It will install the base system and user's selection of packages to an install target and recover useful information like emails, bookmarks, etc from previous installed system wherever possible.

Packages

A "package" is a file that is formed in order application software to be independently and easily added to and removed from the system. Packages Project is responsible for determining the packages that Uludag will include, and forming the packages from the source code and in line with Uludag's goals.
