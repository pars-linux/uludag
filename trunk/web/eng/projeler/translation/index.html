Translating Pardus

Pardus can support your language, and it's easy, yet time consuming to let Pardus speak any language. These are the items you should consider before giving your language support to Pardus. Most of the user interface and help files are in po format, so an editor which can read and write po files (i.e Kbabel or Poedit) will be sufficient.

After installing Pardus, downloading kdesdk package will install Kbabel and place an entry in Pardus menu.

First priority

The 'first priority' items below are considered as the major sources which should be translated at least 80% for a language to be available with Pardus.

  • Localization of YALI (language file)
  • Localization of help and UI files of the following software:
    • Network manager (language file)
    • Package manager (language file)
    • Feedback (language file)
    • Firewall configurator (language file)
    • Tasma (language file)
    • Service manager (language file)
    • User manger (language file)
  • Localization of Mudur (language file)
  • Localization of ÇOMAR (language file)
  • Localization of sysinfo (language file)
  • Localization of Kaptan (language file)
  • Giving initial language support to YALI, i.e adding German KDE files when German language support is chosen.
  • Adding language files for other non-KDE/GNOME applications like scribus, inkscape, etc.

Second priority

These are second priority items. Note that you don't have to translate them, but having them in your language is a big plus.

  • Documents on web page
  • Release notes
  • Summary/description localization of pspec.xml files (around 800 files)

Supported languages

Pardus 1.0 comes with 2 languages (English and Turkish). The following language supports are available for Pardus 1.1, starting from Pardus 1.1 alpha3.

  • English (Maintainer: Görkem Çetin)
  • Turkish (Maintainer: Görkem Çetin)
  • Dutch (Maintainer: Willem Gielen)
