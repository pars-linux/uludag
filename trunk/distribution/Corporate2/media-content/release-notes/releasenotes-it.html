Release Notes

1. Welcome to Pardus Kurumsal 2

Welcome to Pardus Kurumsal 2, the first release of Pardus Corporate series offering a mature enterprise desktop experience to users.

1.1. What is Pardus?

Pardus is an open-source, freely distributable, GPL (General Public License) licensed user-friendly GNU/Linux distribution with many usability, hardware compatibility, stability and security enhancements.
The Pardus project is sponsored by the Center of Research for Advanced Technologies of Informatics and Information Security (BILGEM), an affiliate of the Scientific and Technological Research Council of Turkey (TUBITAK).
The aim of the Pardus distribution is to offer people a stable, reliable, easy-to-use and feature-rich computer experience by providing modern and popular open-source software. The developers are also putting an additional effort to make Pardus run on a wide range of hardware.
The project is named after the latin (and scientific) name for the Anatolian leopard, Panthera pardus tulliana.

1.2. How Can I Try Pardus?

You can try Pardus either by installing it on your computer or running the Live version without touching your hard disk. You can boot Pardus from an optical drive as well as from USB mass storage devices like external hard drives and USB flash drives.
Both the installation and live version images exceeds the size of a regular CD so if you are willing to use an optical drive for booting Pardus, you currently have to use DVD media to burn the images.
Note that the boot order in your system's BIOS software should be set with the optical drive or USB mass storage device as the first device in order to boot from the installation or the live medium. On newer systems, the boot order can also be changed by pressing a key (generally F8 on desktop systems) while booting.

2. Overview

The following sections provides a brief overview of the features offered by Pardus Kurumsal 2.

2.1. Hardware Overview

Pardus Kurumsal 2 supports both 32-bit and 64-bit processor architectures manufactured by Intel and AMD.

2.2. Software Overview

  • Kernel - The Linux kernel shipped with Pardus Kurumsal 2 is 2.6.35 which is the latest kernel branch which will be supported for a long term period.
  • Plymouth - The bootsplash technology used in Pardus Kurumsal 2 is the brand new Plymouth engine.
  • YALI - YALI, the installer of Pardus, gained LVM/RAID and UUID support.
  • KDE - Pardus Kurumsal 2 comes with KDE 3.5.10, the old KDE stack which has proven its stability and reliability. The base packages also contains numerous fixes which will improve the stability of your desktop experience significantly.
  • NetworkManager - GNOME NetworkManager 0.8.2 is the default networking backend in Pardus Kurumsal 2. Users are now able to set up their HSPA/CDMA/VPN networks together with the already supported Ethernet and 802.11 WLAN networks.
  • LibreOffice - LibreOffice, an Open Source personal productivity suite sponsored by the Document Foundation, is the default Office Suite in Pardus Kurumsal 2.

3. What is New?

3.1. Updates in Pardus Technologies

3.1.1. YALI

  • YALI, the installer of Pardus, gained LVM/RAID and UUID support.
  • The user interface is completely redesigned to improve the usability.

3.1.2. Pisi

  • Pisi now internally uses XZ as the default compression algorithm.
  • Binary package names now reflects the architecture and the distribution release for which the package is built.
  • Various speed enhancements has been done in the core components for a better package management experience.

3.1.3. Package Manager

  • The graphical user interface is redesigned to improve the usability.
  • Various speed enhancements has been done for a better graphical package management experience.

3.2. Updates in Core Components

  • Kernel - The Linux kernel shipped with Pardus Kurumsal 2 is 2.6.35 which is the latest kernel branch which will be supported for a long term period.
  • Plymouth - The bootsplash technology used in Pardus Kurumsal 2 is the brand new Plymouth engine.
  • Python - Python 2.6.5 is the default Python interpreter in Pardus Kurumsal 2.
  • Perl - Perl 5.10.1 is the default Perl interpreter in Pardus Kurumsal 2.
  • X.org - The shipped xorg-server version is 1.7.7, improvements to the automatic driver configuration mechanism has been done by our developers.

3.3. Updates in Server Components

  • Web servers - Pardus Kurumsal 2 ships both Apache 2.2.17 and NGINX 0.8.54 HTTP daemon alternatives.
  • Databases - MySQL 5.5.8, PostgreSQL 9.0.3, BerkeleyDB 4.6.21 and SqLite 3.7.3 are among the DBMS alternatives offered in Pardus Kurumsal 2.
  • Mail Servers - Postfix 2.6.5 and Dovecot 1.2.16 are the mail server daemon softwares offered within Pardus Kurumsal 2.

3.4. Desktop Experience

3.4.1. Visual Enhancements

The default icon theme of Pardus Kurumsal 2 is BC Tango which is a Tango derivative specially customized for KDE desktop environment.

3.4.2. Network Management

Pardus Kurumsal 2 uses GNOME NetworkManager as the default networking backend.
With the GNOME networking backend, users are able to easily set up their Ethernet (With support for 802.1x authentication), 802.11 Wireless (With support for WEP/WPA/WPA2/WPA-EAP and 802.1x authentication), EDGE/3G/CDMA and VPN networks.

3.4.3. Office Suite

LibreOffice, an Open Source personal productivity suite sponsored by the Document Foundation, is the default Office Suite in Pardus Kurumsal 2.

4. System Requirements

4.1. Processor (CPU)

You may want to install the 64-bit version of Pardus Kurumsal 2 if your system is equipped with one of the processors listed in the sections below.

4.1.1. Intel processors which support 64-bit architecture

  • Newer versions of Pentium 4, Celeron D, Xeon and Pentium Dual-Core
  • Atom D5xx, N4xx and N5xx
  • All versions of Pentium D, Pentium Extreme Edition, Core 2, Core i7, Core i5 and Core i3 processors
  • Probably newer processors manufactured after the time this document was originally written.

4.1.2. AMD processors which support 64-bit architecture

  • AMD Athlon 64, 64 X2, 64 FX
  • AMD Athlon II
  • AMD Opteron
  • AMD Turion 64
  • AMD Turion 64 X2
  • AMD Sempron
  • AMD Phenom
  • AMD Phenom II
  • Probably newer processors manufactured after the time this document was originally written.

4.2. System memory (RAM)

System memory (RAM) requirements of Pardus Kurumsal 2 depends heavily on your daily workflow so it is very hard to define upper and lower bounds, but we recommend at least 512MB of memory for a smooth desktop experience.

4.3. Hard disk space requirements

4.3.1. Minimum requirements

Pardus Kurumsal 2 requires at least 4GB of free hard disk space for a successful installation. As we don't offer custom package selection during the installation, this minimum size constraint applies to all installations.

4.3.2. Recommended requirements

In order to effectively use Pardus Kurumsal 2 on your system without running low on disk space during your daily workflow, we recommend at least 10GB of free hard disk space.

4.3.3. Note on Swap space

Swap space is used whenever your operating system runs out of physical memory. If you think that your physical memory is big enough for your daily usage, you may skip creating a swap space during installation.
Note that a disk partition reserved for swap space is required if you want to hibernate (Suspend to disk) Pardus. Otherwise the functionality will be disabled.
