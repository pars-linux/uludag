Pardus 2007.2 release notes

1. Welcome to Pardus 2007.2

Welcome to Pardus version 2007.2, a maintenance release for Pardus 2007 after Pardus 2007.1, featuring easy to install and use operating system with providing many useful, compatible and secure development over K Desktop Environment (KDE). This document contains basic differences between Pardus 2007.1 and 2007.2. We kindly invite you to visit www.pardus.org.tr/eng for more information.

1.1 Feedback

You can use bugs.pardus.org.tr, a.k.a Pardus bug tracking system to send a bug, bug fix or any suggestion related to Pardus. We would like to thank our users and developers with their bug reports, related fixes, new package or feature suggestions for this release.

Also, you may use the feedback application available via menu to send your ideas about Pardus over Internet.

2. Basic features

This section describes the features Pardus 2007.2 introduces. Pardus 2007.2 is mainly a bugfix release, translation update and minor extensions to Pardus 2007.1.

2.1 Updated KDE 3.5.7)

Pardus 2007.2 now introduces KDE 3.5.7 for better stability, translations and eye candy for Pardus users. This package is also available for Pardus 2007 and Pardus 2007.1 users through Pardus package manager.

2.2 PEAP-MSCHAPv2 support for Network Manager

Network manager application now comes bundled with PEAP-MSCHAPv2 support. Now Pardus clients can authenticate with wireless devices using this protocol, benefitting from strong encryption possibilities.

2.3 Improved package management system

Some users have complained regarding the speed of search facility in packager manager. Now pisi benefits from a new search method for package names and descriptions. As a result, package searchs are more faster and reliable.

2.4 Package updates

Pardus 2007.2 repository has an ever-increasing number of software, containing 1728 tested and stable packages in this release. Moreover, Pardus users can enjoy to download more than 650 packages by adding contrib repository to Package Manager.

2.5 Boot manager

Pardus 2007.2 introduces a fresh boot manager which can be used to add, modify or delete GRUB entries seen during the boot process.

2.6 Autologin support in YALI

Many users have been asking us a way to quickly start Pardus desktop, and we have implemented it. YALI (Yet Another Linux Installer) now includes autologin support during the installation. That means, you can choose a user which can automatically login to the KDE desktop as system boots.

2.7 Updates and fixes

Jmicron support has been added to Pardus 2007.2. Those having Jmicron controllers are now able to install Pardus 2007.2 without any hassle.

3. Installation requirements

You can run Pardus 2007.2 on an average hardware. We suggest you at least 512 MB RAM (memory) and 1200 MHz processor to get an effective performance. Minimum and recommended hardware requirements for Pardus 2007.2 are mentioned below:
  • 256 MB memory (512 MB recommended)
  • 800 MHz Intel or AMD processor (1200 MHz recommended)
  • At least 4 GB of free hard disk space (10 GB recommended)
